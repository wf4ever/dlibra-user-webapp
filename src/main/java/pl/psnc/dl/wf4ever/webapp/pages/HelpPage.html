How to import data from myExperiment to dLibra?

The import wizard allows you to copy your resources (workflows/files/packs) from myExperiment to dLibra, aggregating them as Research Objects (RO).

The wizard has the following steps:

  1. Start the wizard by pressing the button on the main page. If the button is disabled, make sure that you are registered in dLibra.

  2. Select resources. Choose if you want to import all resources as 1 RO (for example all files and workflows) or every resource as a separate RO (for example all packs). You can later repeat this step many times if you want.

  3. Choose RO names. The proposed names are based on resource titles. RO names are currently also used as RO identifiers, so you may only use alphanumeric characters and the underscore character.

    The RO names have to be unique during the import process. The wizard will not continue if you provide a name that was already used. However, you can provide a name of an existing RO, in which case the imported resources will be added to it.

  4. Go back to step 1 if you want to define more ROs. If you leave the checkbox checked, you will go back to step number 2. If you uncheck it, you will go to the next step.

  5. Press Start. The wizard will create ROs in dLibra and copy selected resources from myExperiment to dLibra. If an error occurs during import, importing the current RO will be interrupted and the wizard will try to import the next RO.

    All status messages will appear in the text area. At the end of the import process a message will appear on the top of the screen.

  6. Your data has been imported.
